# Narro 한국어 메시지 정의서

## 작성 규칙

- 안내 메시지는 `msg_`, 경고 메시지는 `alt_`, 오류 메시지는 `err_` 접두사를 사용한다.
- ID는 배포 후 의미를 변경하거나 재사용하지 않는다. 삭제된 ID도 다른 문구에 다시 할당하지 않는다.
- `%1$s`는 문자열, `%1$d`는 정수 서식 인자다. 인자 순서는 번역 언어에서 변경할 수 있다.
- 파일명과 사용자가 가져온 본문은 번역하거나 임의로 수정하지 않는다.
- 문구 끝의 마침표와 물음표는 Android 문자열 리소스에도 그대로 유지한다.
- 버튼 문구, 화면 제목, TalkBack 전용 설명은 이 문서와 별도의 UI 문자열 리소스에서 관리한다.

## 구현 매핑 원칙

| 접두사 | 기본 표시 방식 | 기본 사용자 행동 |
|---|---|---|
| `msg_` | 인라인 상태, 스낵바, 진행 영역, 알림 | 확인 또는 자동 소멸 |
| `alt_` | 인라인 경고 또는 확인 다이얼로그 | 계속, 취소, 설정 열기 중 선택 |
| `err_` | 오류 배너, 오류 다이얼로그, 전체 오류 화면 | 다시 시도, 다른 파일 선택, 설정 열기 또는 앱 종료 |

- `err_001`과 `err_060`은 구체적인 오류를 분류할 수 없는 최후의 폴백으로만 사용한다.
- 사용자에게 표시하는 ID와 내부 예외·로그 코드를 함께 기록하되 스택 추적이나 파일 경로는 화면에 노출하지 않는다.
- 일시적인 진행 안내는 같은 영역에서 교체하고 스낵바를 연속으로 쌓지 않는다.
- 삭제와 중복 가져오기처럼 데이터에 영향을 주는 작업은 반드시 `alt_` 확인 후 실행한다. 앱 잠금 Off는 설정 화면에 PIN 초기화 결과를 명시하고 즉시 적용한다.
- `err_046`처럼 파일과 DB의 일관성이 깨질 수 있는 오류는 사용자에게 재시도를 제공하고 별도 진단 로그를 남긴다.
